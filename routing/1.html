Routing

The Idea of Routing

Usually if you mess with the URL of the page you’re on with Javascript, the browser is going to make a request for the new page.

The URL is available as window.location. You can set it, if you want, to any URL, like: 


window.location = 'http://mozilla.org';

Go ahead and paste that into your console, press enter, and you’ll see that you end up at mozilla.org. We’ll be here when you back button.

Welcome back. So we’ve seen that changing the URL can be accomplished by Javascript. But of course, when you go to that new page, Javascript “starts over.” So if you do this: 


var important = "something important";
window.location = 'http://mozilla.org';

There’s another way to do this, and it’s called routing. When you “route” with Javascript, you intercept the request to change the URL you’re on, and instead of actually going to some other site, you just run a function of your own choosing. Normally, the point of this is that you can load up some part of the page with content. It has the effect of looking like you’ve gone to a different page, but there’s no actual network request — everything is done by Javascript.

An example will make all this clearer. Let’s say you’ve got three pieces of content: about, contact, and welcome:

Welcome
Welcome to Zombocom!
About
Anything is possible with Zombocom!
Contact
Email us at zombocom@example.com.

Now, on a normal site you might have folders for each of these sections, and inside each folder (about/, contact/, and welcome/) there would be an index.html. So, the following links would work: 


<li><a href="welcome/">welcome</a></li>
<li><a href="about/">about</a></li>
<li><a href="contact/">contact</a></li>

I.e., if you clicked welcome, you’d end up at welcome/index.html which would be a whole HTML page that contained the content “Welcome to Zombocom!” and whatever else.
