Routing

The Idea of Routing

Usually if you mess with the URL of the page you’re on with Javascript, the browser is going to make a request for the new page.

The URL is available as window.location. You can set it, if you want, to any URL, like: 


window.location = 'http://mozilla.org';

Go ahead and paste that into your console, press enter, and you’ll see that you end up at mozilla.org. We’ll be here when you back button.

Welcome back. So we’ve seen that changing the URL can be accomplished by Javascript. But of course, when you go to that new page, Javascript “starts over.” So if you do this: 


var important = "something important";
window.location = 'http://mozilla.org';

When you get to http://mozilla.org, the variable important is gone, never to be seen again. This is kind of a bummer, because if you’re writing an application with Javascript (as opposed to just a plain, static HTML page), then you might want to be able to use URLs as a sort of “navigation” technique.
